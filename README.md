###나만의 메모장 만들기###

3계층 구조(Layered Architecture)이해
계층 (Layer) | 역할 (비유) |  실제 기능
Controller  | 점원 (카운터)| "사용자의 요청(URL)을 받고, 최종 결과를 응답함"
Service     | 주방장 (요리)| "핵심 비즈니스 로직(데이터 가공, 검사 등)을 처리함"
Repository  | 창고 관리자  |  DB에 접근해서 데이터를 저장하거나 꺼내옴

1단계 도메인(Entity) 설계
메모에 필요한 최소한의 정보만 정의 
- id(고유 번호)
- title(제목)
- content(내용)
- createdAt(작성 시간)

2단계 계층별 구현 순서
처음에는 역순으로 올라오는 것이 흐름 잡기에 편하다
1. Repository : JpaRepository를 상속받아 인터페이스만 만든다(DB 연결 통로 확보)
2. Service : "메모를 저장한다","전체 메모를 가져온다" 같은 기능을 메서드로 만든다. 
3. Controller : @PostMapping("/notes") 같은 경로를 설정하고, 서비스의 메서드를 호출한다.
